# 代码审查摘要（2025-12-07）

## 高优先级
1. 个税专项附加扣除校验仍按年度上限 96,000 元（`SalaryTaxPage.ets`）  
   - 现已改为单月预扣口径，但校验上限仍用年度值，月度场景下应以 8,000 元/月或明确提示当前按年度校验，避免误判。

## 中优先级
2. 税务日历截止日缺省时展示不一致（`TaxCalendarPage.ets`）  
   - 当未找到节点时 `nextDeadline` 显示 “暂无”，但 `nextDeadlineDays` 仍为 0，可能导致 UI 误解为“今天截止”。建议在无节点时同时置为 0/空且明确文案。
3. 税务日历节点仍为静态全国口径（`TaxCalendarPage.ets`）  
   - 目前按固定列表计算，未区分地区/口径。建议至少在 UI 提示“按全国通用口径，仅供参考”，或支持地区配置。

## 低优先级
4. 社保与公积金封顶使用同一数值（`SalaryTaxPage.ets`）  
   - 目前两者共用 `currentCityMaxBase`，若后续需要区分社保/公积金封顶值需拆分存储字段。

## 建议的下一步
- 高：调整专项附加扣除校验为月度上限或在单月模式下按月度校验，避免用户输入 10,000 元/月仍被判定通过。  
- 中：无可用截止日时的显示与计数统一；在税务日历文案提示“全国通用口径，仅供参考”，后续可支持地区化配置。  
- 低：如需支持社保/公积金不同封顶值，预留独立字段并兼容当前存量数据。
